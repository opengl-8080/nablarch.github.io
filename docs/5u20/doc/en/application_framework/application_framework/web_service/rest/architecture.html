3.1.1. Architecture Overview¶

Table of Contents

  • Structure of RESTful web service
  • Process flow of RESTful web service
  • Handlers used by RESTful web service
    • Standard handler configuration

Nablarch provides the functions to create RESTful web service using the business action of a web application (JAX-RS support) in the same way JAX-RS resource class is created.

JAX-RS support is based on web applications of Nablarch. Therefore, JAX-RS cannot use Servlet resource injection or CDI using @Context annotations, which can be used in JAX-RS. The following annotations are available for JAX-RS support.

  • Produces (specify media type for the response)
  • Consumption (specify the media type of the request)
  • Valid (runs BeanValidation for the request)

For comparison of functions between JSR339 and JAX-RS support , see Function Comparison of JAX-RS Support /JSR339/HTTP Messaging.

Important

JAX-RS support does not provide client-side functions. If you need to use a JAX-RS client, use a JAX-RS implementation (such as Jersey or RESTEasy).

3.1.1.1. Structure of RESTful web service¶

It has the same structure as the Nablarch web application. For more information, see Structuring a web application.

3.1.1.2. Process flow of RESTful web service¶

The process flow of RESTful web service, from processing a request to returning a response, is shown below.

  1. Web Front Controller (implementation class of javax.servlet.Filter) receives a request.
  2. Web Front Controller delegates the processing of the request to a handler queue (handler queue).
  3. DispatchHandler configured in the handler queue specifies the action class to be processed based on the URI and adds it to the end of the handler queue.
  4. The action class executes business logic using a form class and an entity class.
    For more information on each class, see Responsibility Assignment of RESTful Web Service.
  5. The action class creates and returns a form class and HttpResponse that shows the processing result.
  6. The HTTP response handler (JaxRsResponseHandler) in the handler queue converts the HttpResponse into a response to be returned to the client, and returns the response to the client.
    If the processing result of the action class is a form class, it will be converted into HttpResponse by BodyConvertHandler.
    The format of the HttpResponse body to be converted is the media type configured in the action class.

3.1.1.3. Handlers used by RESTful web service¶

Nablarch provides several handlers required for building RESTful web service. Build the handler queue in accordance with the requirements of the project (a custom handler will have to be created for the project depending on the requirements)

For details of each handler, refer to the link.

Handlers that convert request and response
  • JAX-RS Response Handler
  • Request Body Conversion Handler
Handlers associated with database
  • Database Connection Management Handler
  • Transaction Control Handler
Handlers for request verification
  • JAX-RS BeanValidation Handler
  • CSRF Token Verification Handler
Error handling handler
  • Global Error Handler
Other handlers
  • Handler to link request URI and action
  • Health Check Endpoint Handler

3.1.1.3.1. Standard handler configuration¶

When building RESTful web service application in Nablarch, the minimum required handler queue is as below: With this as the base, add standard handlers of Nablarch or custom handlers created in the project according to the project requirements.

Minimum handler configuration¶
No. Handler Request process Response process Exception handling
1 Global Error Handler     Outputs the log for a runtime exception or error.
2 JAX-RS Response Handler   Writes the response. Generates responses for exceptions (errors), writes and outputs the responses to logs.
3 Database Connection Management Handler Acquires DB connection. Releases the DB connection.  
4 Transaction Control Handler Begin a transaction. Commits the transaction. Rolls back a transaction.
5 Handler to link request URI and action Determine the action (method) to call based on the request path.    
6 Request Body Conversion Handler Converts request body to a form class that is accepted by an action. Converts the form content of the action process results to a response body.  
7 JAX-RS BeanValidation Handler Executes the validation for the form class converted by No6.    

Tip

Handlers configured after Handler to link request URI and action are not configured directly in the handler queue but for Handler to link request URI and action.

If JAX-RS Adapter is used, Request Body Conversion Handler and JAX-RS BeanValidation Handler are added to the handler queue.

To configure handlers other than Request Body Conversion Handler and JAX-RS BeanValidation Handler or to increase the media types that are supported, build a handler queue referring to the configuration example or implementation of JAX-RS Adapter.

<component name="webFrontController" class="nablarch.fw.web.servlet.WebFrontController">
  <property name="handlerQueue">
    <list>
      <!-- Handler of the previous stage is omitted -->

      <!-- Configure a handler to link request URIs and actions -->
      <component name="packageMapping" class="nablarch.integration.router.RoutesMapping">
        <!-- Configuration values other than handlers are omitted -->
        <property name="methodBinderFactory">
          <component class="nablarch.fw.jaxrs.JaxRsMethodBinderFactory">
            <property name="handlerList">
              <list>
                <!--
                Configure the handler queue after the handler that links the request URI to the action
                *Configuration values of each class are omitted
                -->
                <component class="nablarch.fw.jaxrs.BodyConvertHandler">
                  <!-- Configure the supported media types of converters -->
                </component>
                <component class="nablarch.fw.jaxrs.JaxRsBeanValidationHandler" />
              </list>
            </property>
          </component>
        </property>
      </component>
    </list>
  </property>
</component>
